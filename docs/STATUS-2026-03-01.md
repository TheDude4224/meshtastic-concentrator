# Project Status — 2026-03-01
## FreedomFi Gateway + SenseCAP IoT Hotspot as Meshtastic Mesh Nodes

---

## What We Built

Converted two pieces of Helium mining hardware into 8-channel Meshtastic 
"super nodes" using ChirpStack Concentratord + a custom Python bridge.

Standard Meshtastic devices use single-channel radios (SX1262/SX1276).
These nodes use multi-channel concentrator cards that listen on **8 channels 
simultaneously**, giving dramatically better packet capture vs a standard node.

## Known Limitations / In Progress

⚠️ **TX not implemented** — The bridge's `send` command increments a counter
but the actual `send_downlink()` call to concentratord is stubbed out.
Messages are NOT transmitted over the air yet. Requires implementing:
- Meshtastic MeshPacket protobuf encoding
- AES-CTR encryption with channel key
- Proper concentratord downlink command format

⚠️ **RX not confirmed** — rx_count = 0 on both nodes. No other Meshtastic
devices in range to confirm reception is working end-to-end.

⚠️ **No inter-node communication yet** — oryahnoc and meshnode-1 cannot
hear each other in testing (suspected: TX stub means neither is actually
emitting RF despite concentratord being live)

## Key Technical Discoveries

1. **ChirpStack apt repo has no concentratord package** for Ubuntu 24.04 / 
   Debian Trixie — must build from source

2. **Must use brocaar/sx1302_hal fork** (tag V2.1.0r9), NOT the official 
   Lora-net/sx1302_hal — the official fork is missing `lgw_i2c_set_path` 
   and `lgw_i2c_set_temp_sensor_addr` required by concentratord v4.6+

3. **RAK5146 model name** in config is `rak_5146` (underscore), not `rak5146`

4. **Channels must be explicit** — the `[gateway.concentrator]` section with 
   `multi_sf_channels` array is required; region alone does not populate channels

5. **FreedomFi RAK5146** shows as `34e1:1002` (USB Virtual COM) when healthy.
   `0483:df11` = DFU/bricked mode — replace the card

6. **Build time:** ~5 min on J1900, ~7 min native on RPi4
